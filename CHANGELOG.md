### Fixed
- Listing/search/news request acceptance + response kind tagging: `ListingClient.LoadPopular/LoadRecent/LoadNews` and `SearchClient.SubmitSearch` now return `bool` (true on accept, false on busy/invalid), and the controller updates `_currentTab` / `_activeTabIndex` / state only after acceptance. Each client also tags the response with the request kind (`"popular"` / `"recent"` / `"news"` / `"search"`); the controller passes the kind to `RenderResultList(kind)` instead of using mutable `_currentTab` at render time, and discards stale responses whose kind no longer matches the active tab. With Search and Listing on independent `_isLoading` flags they can race; without this fix a Popular response could render with empty title/body fields after the user clicked News mid-flight (#35 review)

### Changed
- Removed `#TabSearch` button. ### Fixed
- Search URL prefix mismatch on submit (#23 polish, user-reported VR test bug + Play Mode test follow-up): two-step fix.
  - **Initial issue**: testing-chamber scene's `#SearchInputField` had empty/stale `m_Text`, causing `SubmitSearch` to always fail prefix validation. PoolGenerator now writes the prefix into `SearchClient._expectedUrlPrefix` and `VRCUrlInputField.m_Text` whenever Generate is run; PoolGeneratorWindow gains an auto-resolved SearchClient ObjectField.
  - **Follow-up bug discovered in Play Mode**: even after PoolGenerator wrote `m_Text`, the Play Mode test still failed because `VRCUrlInputField` is **not** a `TMP_InputField` subclass — it extends `Selectable` directly and reads its URL from a child `m_TextComponent` (`UnityEngine.UI.Text` legacy or `TMP_Text`) at runtime. Setting `m_Text` alone leaves the TextComponent empty, and on Play Mode start `VRCUrlInputField` overwrites `m_Text` with the empty TextComponent value. Fix: `PoolGenerator.GenerateAll` now syncs **3 locations** — `_expectedUrlPrefix` + `VRCUrlInputField.m_Text` + the bound TextComponent's `m_Text` — via SerializedProperty.
  - The runtime error message also embeds the expected prefix verbatim and instructs the user to retype, since `VRCUrlInputField.text` setter is not exposed to Udon (§12 #4) and runtime auto-restore is impossible.
- `_AutoLoadPopular` race: if the user clicked a different tab during the 2-second auto-load delay, the delayed callback still called `RequestPopular(0)` and overwrote the user's tab selection (Popular tab snapped back to active blue, Recent went to inactive surface). `_AutoLoadPopular` now early-returns when `_activeTabIndex != -1`, so any user interaction before the delay elapses suppresses the auto-load (#23 Phase A user-reported VR test bug)
- `#SelectButton` white film over thumbnails: ResultRow's full-row click area was the last sibling and rendered above `#Thumbnail`, so its `Image.color` (white α≈0.08 after Button.colors normal multiply) showed as a faint white film over the thumbnail image. Move `#SelectButton` to sibling index 0 (= behind everything) and set `#Thumbnail` (RawImage) and `#Name`/`#Owner`/`#TrackCount` (TMP) `raycastTarget=false` so clicks pass through to the now-backgrounded button (#23 polish, user-reported)
- DetailView lacked the surface-tint card backgrounds that SearchView's elements had: cover row was bare absolute-positioned text, scroll area BG was transparent (made so in PR #34 to avoid washing out `UI_RoundedPanel` track cards). Add `#CoverBg` Image (UI_RoundedPanel Sliced + surface tint) behind the cover row and revert the Scroll View Image to surface tint, paired with the track template restyle below so the whole scroll area reads as one tracks list card. Both wired into `_surfacePanels[]` (#23 polish)
- Track Title truncation cut text well before the rect edge because `_trackTitleMaxChars` was a char count tuned for all-Japanese worst-case width (632px / fontSize 28 ≈ 22 full-width chars). Mixed/English content left a lot of empty space. Switch the Title to a `VerticalLayoutGroup` + `ContentSizeFitter` + `LayoutElement` setup with TMP wrap=true so titles wrap to 2 (or rarely 3) lines and each track row's height auto-fits the wrapped content. Per follow-up user request, `RenderTrackList` no longer truncates the title at all — raw title text is passed to TMP and the layout system handles overflow via wrapping + variable cell height. The serialized `_trackTitleMaxChars` field is kept (default 0 = no cap) for backward-compat but is no longer read by the renderer. `RenderTrackList` no longer manually positions clones or sets parent sizeDelta — VLG and CSF do it (#23 polish, user-reported)
- TrackTemplate had rounded corners and no vertical spacing between clones (Phase A-4 used `UI_RoundedPanel` sliced + `h * i` manual stacking). Set the BG `Image.sprite = null` for sharp-corner flat rectangles, and let the new `VerticalLayoutGroup` on `#TrackListContent` insert 4px gaps between clones (#23 polish, user-reported)

### Changed
- `#DetailView/Scroll View` vertical extent expanded (#23 polish, user-requested): RectTransform updated from anchoredPos `(0, -85.5)` sizeDelta `(-40, -677)` (height 347 px) to anchoredPos `(0, -101.5)` sizeDelta `(-40, -645)` (height 379 px, +32 px). Bottom edge now sits 16 px above `#UrlField` top (testing-chamber `#UrlField` top measured at canvas y=-307), using the previously empty space below the track list. Visible tracks per view increases by ~1 row. SearchView Scroll View height (780) is unchanged; DetailView keeps cover row + UrlField intact above/below. `docs/unity-architecture.md` §13.6 step 10 documents the formula for scene-specific UrlField positions: `sizeDelta.y = (88 - (UrlField_top_canvas_y + 16)) - 1024`, `anchoredPos.y = (88 + (UrlField_top_canvas_y + 16)) / 2` (canvas h=1024, Scroll View top y=+88 kept; verify: UrlField_top=-307 → sizeDelta.y=-645, anchoredPos.y=-101.5). Hierarchy diagram (L606) and step 10 — previously inconsistent (`y=-26 sd=-573` vs `y=-85.5 sd=-677`) — are now both synced to the new value.
- `#TabNews` is now `SetActive(false)` per user request — the News tab UI shipped in PR #35 is hidden in this polish PR, and the `#TabRow` HorizontalLayoutGroup redistributes Popular/Recent to 50% each. The implementation is intentionally retained (`OnTabNews` / `RequestNews` / `LoadNews` / news-mode `RenderResultList` / `_tabNewsBg` / `_newsUrl` / `PoolGenerator` news bake) so re-enabling the tab is a single `SetActive(true)` change. Server endpoint (vhub-playlist#97) is unchanged (#23 polish, user-requested)
- `OnSelectResultByIndex` desync: `_pendingOwnerName` and `_pendingYtThumbIndex` were updated before `PlaylistResolver.Resolve(...)` accepted the request, so a second row click during an in-flight resolve could overwrite the pending metadata while the resolver kept loading the first playlist. `Resolve` now returns `bool` (true when accepted, false when busy / invalid), and the controller updates the pending fields only on `true`. Validation also runs before any pending mutation, so invalid rows can no longer briefly clear the previous pending values. Phase A-4 made this much more visible because the cover thumbnail is large (#34 review)
